## Component Inventory

This phase adds one inline UI section to `TestConfig.tsx`. No new component files.

### Module Selector Section

**Placement:** Between the "Configuracao de carga" fieldset (`grid-cols-2 gap-3 mb-4` at line 309) and the "Configuracoes Avancadas" collapsible (`div.mb-4` at line 368).

**Visibility condition:** Render ONLY when at least one operation in `config.operations` has a name matching `MISTERT_MODULE_METADATA[].name`. This gates the selector to MisterT-style operations only.

**Structure:**

```
<fieldset className="mb-4">
  <legend className={labelClass}>            -- "Modulos do Teste"
    <LayoutGrid icon />                       -- lucide icon
    <InfoTooltip />                           -- explanation tooltip
  </legend>

  <div className="flex items-center ...">    -- toggle link row
    <button>Selecionar Todos / Limpar</button>
    <span>N de 7 selecionados</span>          -- selection counter
  </div>

  <div className="grid grid-cols-3 gap-x-4 gap-y-2">  -- checkbox grid
    {MISTERT_MODULE_METADATA.map(module =>
      <label>                                 -- checkbox + label
        <input type="checkbox" sr-only />
        <div> visual checkbox box </div>
        <span> module.name </span>
        <span> module.code badge </span>
      </label>
    )}
  </div>

  {allUnchecked &&                            -- warning when 0 selected
    <div role="status"> warning text </div>
  }
</fieldset>
```

### Checkbox Item (inline, not a separate component)

Reuses the exact visual pattern from `WelcomeOverlay.tsx` lines 251-279.

**States:**

| State | Visual |
|-------|--------|
| Unchecked | 18x18px rounded-md box, `border-sf-border bg-sf-surface`, no checkmark |
| Checked | 18x18px rounded-md box, `bg-sf-primary border-sf-primary`, white SVG checkmark |
| Focus (keyboard) | 2px ring `ring-sf-primary/50`, 1px offset `ring-offset-sf-bg` |
| Hover (label) | Label text shifts from `text-sf-text` to slightly brighter (via `group-hover`) |
| Disabled | Not applicable -- all modules are always toggleable |

**Checkbox SVG checkmark (exact match to WelcomeOverlay):**
- viewBox: `0 0 12 12`
- Path: `M2.5 6L5 8.5L9.5 3.5`
- strokeWidth: 1.75
- strokeLinecap: round
- strokeLinejoin: round
- Color: white (`text-white`)

## Interaction Contract

### Checkbox Toggle

| Trigger | Behavior |
|---------|----------|
| Click checkbox or its label | Toggle that module on/off. Calls `updateModuleSelection()` with rebuilt operations array |
| Keyboard Space/Enter on checkbox | Same as click (native input[type=checkbox] behavior via sr-only) |
| Toggle preserves activePreset | `updateModuleSelection` does NOT clear `activePreset` (D4) |
| Toggle rebuilds from template | Full template rebuilt from `buildMistertOperations(currentBaseUrl)`, filtered by checked names, preserving template order |

### Select All / Clear Toggle

| Trigger | Behavior |
|---------|----------|
| Click "Selecionar Todos" | All 7 modules checked. Full template operations applied |
| Click "Limpar Selecao" | All 7 modules unchecked. Only infra ops (0-2) remain |
| Toggle label changes | Shows "Limpar Selecao" when all 7 checked, "Selecionar Todos" otherwise |

### Section Visibility

| Condition | Behavior |
|-----------|----------|
| MisterT operations detected in config | Module selector section visible |
| Non-MisterT preset loaded | Module selector section hidden (no matching operation names) |
| No operations in config | Module selector section hidden |
| Preset applied via PresetModal | Section appears if MisterT ops, checkboxes reflect operations present |
| Preset saved via SavePresetDialog | Current module selection saved as-is in operations[] |

### Zero-Module Warning

| Condition | Behavior |
|-----------|----------|
| All 7 modules unchecked | Warning text appears below the grid: "Nenhum modulo selecionado -- o teste executara apenas login e menu." |
| At least 1 module checked | Warning hidden |
| Warning does NOT block test start | User can run login-only test (valid scenario) |

## Layout Specification

### Grid Dimensions

| Property | Value | Rationale |
|----------|-------|-----------|
| Columns | 3 (`grid-cols-3`) | Labels max ~14 chars ("Faturamento"). Form is max-w-2xl (672px). 3 cols = ~200px each, ample for label + code badge |
| Column gap | 16px (`gap-x-4`) | Matches standard md spacing for clear visual separation between columns |
| Row gap | 8px (`gap-y-2`) | Compact vertical rhythm, same as environment button grid gap |
| Grid items | 7 checkboxes | 3 cols x 3 rows = 9 cells. Last row has 1 item, 2 cells empty |

### Section Vertical Rhythm

```
[Configuracao de carga fieldset]   mb-4 (16px)
[Module Selector fieldset]         mb-4 (16px)
  legend                           mb-2 (8px)
  toggle link row                  mb-2 (8px)
  checkbox grid                    gap-y-2 (8px between rows)
  zero-module warning (if shown)   mt-2 (8px)
[Configuracoes Avancadas button]   mb-4 (16px)
```

### Checkbox Item Layout

Each checkbox item is a horizontal flex row:

```
<label className="flex items-center gap-2.5 cursor-pointer group select-none">
  [18x18 checkbox box]  [module name text-sm]  [code badge text-xs font-mono]
</label>
```

- Checkbox box: `w-[18px] h-[18px]` fixed
- Module name: `text-sm text-sf-text` flex-shrink allowed
- Code badge: `text-xs text-sf-textMuted font-mono px-1.5 py-0.5 bg-sf-surface rounded ml-auto` -- right-aligned within column

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Checkbox semantics | Native `<input type="checkbox">` with `sr-only` class. Screen readers announce checked/unchecked state natively |
| Label association | Each `<label>` wraps both the hidden input and visible text -- implicit association |
| Keyboard navigation | Tab moves between checkboxes. Space toggles. Standard browser behavior via native inputs |
| Focus indicator | `peer-focus-visible:ring-2 peer-focus-visible:ring-sf-primary/50 peer-focus-visible:ring-offset-1` |
| Section grouping | `<fieldset>` with `<legend>` groups all checkboxes semantically |
| Warning announcement | Zero-module warning uses `role="status"` for polite live announcement |
| Toggle link | `<button type="button">` with descriptive text. Not a link (`<a>`) since it performs an action |
