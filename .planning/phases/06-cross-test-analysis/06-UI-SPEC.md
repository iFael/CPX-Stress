## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f1117` (sf-bg) | Page background |
| Secondary (30%) | `#1a1d27` (sf-surface) | Cards, table rows, chart container, test selector panel |
| Accent (10%) | `#6366f1` (sf-primary) | Selected test checkboxes, "Comparar" CTA button, active states |
| Destructive | `#ef4444` (sf-danger) | Not used in this phase (read-only screen, no destructive actions) |

### Accent reserved for:
- Primary CTA button ("Comparar Testes")
- Selected/active checkbox state
- Active sidebar nav item highlight
- Chart tooltip active dot stroke

### Semantic colors for chart bar palette:
Each selected test gets a unique bar color from this ordered palette (max 5 tests):

| Slot | Color | Token |
|------|-------|-------|
| Test 1 | `#6366f1` | sf-primary (indigo) |
| Test 2 | `#22d3ee` | sf-accent (cyan) |
| Test 3 | `#22c55e` | sf-success (green) |
| Test 4 | `#f59e0b` | sf-warning (amber) |
| Test 5 | `#3b82f6` | sf-info (blue) |

### Degradation indicator:
When an operation's error count increases between consecutive tests (ordered by date), the table cell background uses `bg-sf-danger/10` and text uses `text-sf-danger`. The `TrendingUp` icon from lucide-react (16x16, `text-sf-danger`) appears inline next to the count.

When errors decrease, the cell uses `bg-sf-success/10` and `text-sf-success` with `TrendingDown` icon.

Neutral (no change or non-consecutive): default `text-sf-text` with no background.

Source: existing color tokens in `tailwind.config.mjs`, chart colors from `TestProgress.tsx` and `TestResults.tsx`.

### 2. TestSelectorList (inline in CrossTestAnalysis)

- Source data: `window.stressflow.history.list()` via Zustand `history` state
- Each row: checkbox + health indicator dot + URL (truncated) + date + error count badge + VU count
- Search input: filters by URL text (same pattern as HistoryPanel)
- Minimum 2 selected to enable CTA
- Maximum 5 selectable (disable remaining checkboxes when 5 reached)
- Sorted by date descending (most recent first)
- Card container: `bg-sf-surface border border-sf-border rounded-xl p-4`
- Row: `flex items-center gap-2 px-4 py-2 rounded-lg hover:bg-sf-surfaceHover transition-colors cursor-pointer`
- Selected row: `bg-sf-primary/5 border-sf-primary/20`
- Checkbox: 16x16, `rounded border-sf-border`, checked state `bg-sf-primary border-sf-primary`

### 3. ComparisonTable (inline in CrossTestAnalysis)

- Rows: one per distinct operation name across all selected tests
- Columns: "Operacao" (sticky left) + one column per selected test
- Column header: date + VU count (e.g., "06/04 15h — 25 VUs")
- Cell content: error count (font-mono) + optional trend icon + percentage delta
- Sort: by total errors descending across all selected tests (operation with most total errors at top)
- Zero errors cell: `text-sf-textMuted` with em-dash "—"
- Container: `bg-sf-surface border border-sf-border rounded-xl overflow-hidden`
- Header row: `bg-sf-bg text-sf-textSecondary text-xs font-bold`
- Body rows: `divide-y divide-sf-border hover:bg-sf-surfaceHover/50 transition-colors`

### 4. ComparisonChart (inline in CrossTestAnalysis, uses Recharts BarChart)

- Recharts `<BarChart>` with `<Bar>` components, one per selected test
- X-axis: operation names (truncated to 12 chars + ellipsis if needed)
- Y-axis: error count
- Bars grouped by operation, color assigned from the 5-color palette above
- Chart height: 280px (fixed, same pattern as MetricsChart at 220px but taller for grouped bars)
- Container: `bg-sf-surface border border-sf-border rounded-xl p-4`
- Title: `text-sm font-bold text-sf-textSecondary mb-4` — "Erros por Operacao"
- Custom tooltip: follow `MetricsChart.tsx` tooltip pattern — `bg-sf-surface border border-sf-border rounded-[10px] shadow-[0_4px_24px_rgba(0,0,0,0.4)]`
- Custom legend: follow `MetricsChart.tsx` CustomLegend pattern — `flex justify-center flex-wrap gap-4 pt-1` with colored dots and test labels
- Grid: `strokeDasharray="3 3" stroke="#1e2130" vertical={false}`
- Axis: `stroke="#64748b" fontSize={11} tickLine={false}`

Source: `MetricsChart.tsx` THEME constants and Recharts patterns.

## Interaction States

### Test Selector
| State | Visual |
|-------|--------|
| Default (0-1 selected) | CTA button disabled: `opacity-50 cursor-not-allowed bg-sf-primary/50` |
| 2+ selected | CTA button enabled: `bg-sf-primary hover:bg-sf-primaryHover text-white` |
| 5 selected (max) | Remaining unchecked rows show disabled checkbox with `opacity-40`, tooltip: "Maximo de 5 testes para comparacao" |
| Loading history | Skeleton shimmer in 3 rows using `animate-shimmer` |
| Search active | Clear button (X icon) appears in search input (same pattern as HistoryPanel) |

### Comparison View
| State | Visual |
|-------|--------|
| Loading aggregation | Spinner centered in table area: `Loader2 animate-spin` + "Carregando dados..." |
| Data loaded | Table and chart render with `animate-fade-in` entrance |
| All zeros (no errors in any selected test) | Special state: success message in place of table/chart |

### Sidebar Navigation
| State | Visual |
|-------|--------|
| Active | `bg-sf-primary/10 text-sf-primary font-bold` (matches existing nav pattern) |
| Inactive | `text-sf-textSecondary hover:bg-sf-surfaceHover hover:text-sf-text` |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Test selector list navigable with Tab + Space to toggle checkboxes |
| Screen reader | `role="region" aria-label="Analise cross-test"` on page wrapper |
| Table semantics | Standard `<table>` with `<thead>`, `<tbody>`, `<th scope="col">` and `<th scope="row">` |
| Chart alt text | `aria-label` on chart container: "Grafico de barras comparando erros por operacao entre {n} testes" |
| Trend indicators | `aria-hidden="true"` on trend icons, sr-only text with percentage delta |
| Focus ring | `sf-focus-ring` utility on all interactive elements (checkboxes, CTA button, search input) |
| Live region | `aria-live="polite"` on selection counter text |
