## Component Specifications

### CredentialsSettings.tsx

**Layout structure:**

```
+-----------------------------------------------------+
|  [Settings icon]  Configuracoes          (heading)   |
+-----------------------------------------------------+
|                                                       |
|  +--- Credenciais MisterT (sf-card) ---------------+ |
|  |                                                   | |
|  |  [User icon]  Usuario        [Configurado / --]  | |
|  |  +-------------------------------------------+   | |
|  |  |  Usuario do MisterT            (input)    |   | |
|  |  +-------------------------------------------+   | |
|  |                                                   | |
|  |  [Lock icon]  Senha          [Configurado / --]  | |
|  |  +-------------------------------------------+   | |
|  |  |  ********               [Eye toggle]      |   | |
|  |  +-------------------------------------------+   | |
|  |                                                   | |
|  |  +-------------------------------------------+   | |
|  |  |       [Save icon]  Salvar Credenciais     |   | |
|  |  +-------------------------------------------+   | |
|  |                                                   | |
|  |  Armazenado em: C:\Users\...\stressflow\.env     | |
|  +---------------------------------------------------+ |
|                                                       |
+-------------------------------------------------------+
```

**Visual specifications:**

- Container: `max-w-2xl mx-auto animate-slide-up` (matches TestConfig container)
- Page heading: `text-lg font-semibold text-sf-text` with Settings icon (`w-5 h-5`) in `flex items-center gap-2`
- Card: `bg-sf-surface border border-sf-border rounded-xl p-6` (standard sf-card pattern)
- Section heading inside card: `text-sm font-semibold text-sf-text mb-4` with KeyRound icon
- Input fields: Reuse `inputBaseClass` from TestConfig pattern: `w-full px-4 py-2.5 bg-sf-surface border border-sf-border rounded-xl text-sf-text focus:outline-none focus:ring-2 focus:ring-sf-primary/30 focus:border-sf-primary transition-all`
- Input labels: Reuse `labelClass`: `flex items-center gap-2 text-sm text-sf-textSecondary mb-2`
- Status indicators per field: Inline `flex items-center gap-1.5 text-xs` aligned right in the label row
  - Configured: `Check` icon (w-3.5 h-3.5) `text-sf-success` + "Configurado" `text-sf-success`
  - Not configured: `Minus` icon (w-3.5 h-3.5) `text-sf-textMuted` + "Nao configurado" `text-sf-textMuted`
- Password eye toggle: Button inside the input container (absolute positioned right), icon `Eye` / `EyeOff` (w-4 h-4) `text-sf-textMuted hover:text-sf-textSecondary`
- Save button: `w-full py-2.5 bg-sf-primary hover:bg-sf-primaryHover text-white font-semibold rounded-xl text-sm transition-all flex items-center justify-center gap-2 mt-4` with Save icon (w-4 h-4)
- Save button disabled state: `disabled:opacity-50 disabled:cursor-not-allowed` -- disabled when BOTH fields are empty
- .env path info: `text-xs text-sf-textMuted mt-4` displayed below the save button

**Interaction contract:**

| Interaction | Behavior |
|-------------|----------|
| Page load | Fields always start EMPTY (security: never pre-fill with saved values). Status indicators load from Zustand `credentialStatus`. |
| Type in field | Standard controlled input. Values exist ONLY in component-local `useState`. |
| Toggle password visibility | Click eye icon toggles between `type="password"` and `type="text"`. Icon switches between `EyeOff` (hidden) and `Eye` (visible). |
| Click "Salvar Credenciais" | Button shows loading state (Loader2 spinner). IPC `credentials:save` is called. On success: clear fields, show success toast, refresh credential status in Zustand. On error: show error toast. |
| Empty field on save | SKIP that field -- do not overwrite existing credentials with empty string. Only non-empty fields are sent to IPC. |
| Both fields empty | Save button is disabled. Tooltip or visual cue: button opacity 50%. |
| After successful save | Fields reset to empty. Status indicators update to "Configurado" for saved fields. Toast: "Credenciais salvas com sucesso!" |
| Keyboard | Tab navigation through fields. Enter on focused button triggers save. |

**Animation:** `animate-slide-up` on container (0.35s cubic-bezier(0.16, 1, 0.3, 1)), matching TestConfig entrance.

### CredentialAlert.tsx

**Layout structure:**

```
+------------------------------------------------------------------+
| [AlertTriangle]  Credenciais MisterT nao configuradas             |
|                  Configure usuario e senha para       [Configurar]|
|                  executar testes autenticados no                   |
|                  MisterT ERP.                                     |
+------------------------------------------------------------------+
```

**Visual specifications:**

- Container: `mb-4 p-4 bg-sf-warning/10 border border-sf-warning/30 rounded-xl flex items-start gap-3`
- Icon: `AlertTriangle` (w-5 h-5) `text-sf-warning shrink-0 mt-0.5`
- Heading: `text-sm font-semibold text-sf-warning`
- Body: `text-sm text-sf-warning/80 mt-1`
- Action button: `text-sm text-sf-warning hover:text-sf-warning/80 font-semibold underline shrink-0`
- ARIA: `role="alert"` on the container

**Interaction contract:**

| Interaction | Behavior |
|-------------|----------|
| Visibility | Rendered in TestConfig when `credentialStatus` has any value set to `false`. NOT rendered when `credentialStatus` is `null` (still loading) or all values are `true`. |
| Click "Configurar" | Calls `setView("settings")` to navigate to the settings screen. |
| Dismissal | NOT dismissible. Persists until all required credentials are configured. |

**Source:** Research Pattern 4 (02-RESEARCH.md lines 217-248), existing error alert pattern (TestConfig.tsx lines 426-446).

## State Contract

| State Field | Type | Location | Purpose |
|-------------|------|----------|---------|
| `credentialStatus` | `CredentialStatus \| null` | Zustand store | Boolean map of required credential keys. `null` = not yet checked. |
| `user` | `string` | CredentialsSettings local `useState` | Username input value. NEVER in Zustand. Cleared after save. |
| `pass` | `string` | CredentialsSettings local `useState` | Password input value. NEVER in Zustand. Cleared after save. |
| `showPassword` | `boolean` | CredentialsSettings local `useState` | Eye toggle state. Default: `false`. |
| `isSaving` | `boolean` | CredentialsSettings local `useState` | Loading state for save button. |

**Security invariant:** Credential values exist ONLY in component-local state (cleared after save) and IPC transit. They NEVER enter Zustand, localStorage, or any globally-accessible renderer state.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Screen reader: alert banner | `role="alert"` on CredentialAlert container |
| Screen reader: form | `role="form"` on CredentialsSettings form container with `aria-label="Configuracao de credenciais MisterT"` |
| Screen reader: status | Status indicators use `aria-label` with full text ("Credencial de usuario configurada" / "Credencial de usuario nao configurada") |
| Screen reader: toast | `aria-live="polite"` on toast container (already implemented in Toast.tsx) |
| Keyboard: focus ring | All interactive elements use existing `sf-focus-ring` pattern or `focus-visible:ring-2 focus-visible:ring-sf-primary` |
| Keyboard: nav | Settings nav item follows existing arrow-key navigation in Sidebar |
| Keyboard: eye toggle | Button with `aria-label="Mostrar senha"` / `aria-label="Ocultar senha"` |
| Skip nav | Existing skip link in Layout.tsx targets `#main-content` (already covers settings view) |
| Color contrast | sf-text (#e2e8f0) on sf-bg (#0f1117) = 14.5:1 ratio. sf-textSecondary (#94a3b8) on sf-surface (#1a1d27) = 5.2:1 ratio. Both exceed WCAG AA. |
