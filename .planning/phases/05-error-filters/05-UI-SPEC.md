## Interaction Contracts

### I1: Click-to-Toggle Cards

All three summary cards (Tipo de Erro, Status HTTP, Operacao) share identical toggle behavior:

1. Click an item: sets the corresponding filter state, creating a chip
2. Click the same item again: clears the filter (sets to `undefined`)
3. Click a different item in the same card: replaces the filter value
4. All filter changes reset pagination to page 0

**Source:** Existing ErrorExplorer pattern (lines 148-152, 182-184), RESEARCH.md Pitfall 2

### I2: Period Filter Inputs

1. User selects a datetime in the start input: `filterTimeStart` is set
2. User selects a datetime in the end input: `filterTimeEnd` is set
3. Either input can be set independently (start-only or end-only is valid)
4. Both empty = no period filter applied
5. A period chip appears when either input has a value
6. Clicking the chip X clears both inputs simultaneously
7. Filter changes reset pagination to page 0

**Source:** CONTEXT.md D3, RESEARCH.md Pitfall 5

### I3: Combined Filter Behavior

All filters combine via AND:
- `filterStatusCode` AND `filterErrorType` AND `filterOperationName` AND `timestampStart` AND `timestampEnd`
- The SQL query handles the intersection server-side with pagination
- Summary card counts are NOT affected by other filters (they show total distribution per test)
- Only the table results reflect the combined filters

**Source:** CONTEXT.md D4 ("Todos os filtros combinados via AND"), RESEARCH.md Pitfall 4

### I4: Empty Filtered Results

When combined filters produce zero results:
- Table shows: "Nenhum erro encontrado com os filtros atuais." (existing copy, centered, `text-sf-textMuted`)
- Summary cards remain visible and interactive (they are navigation aids, not filtered views)
- Active filter chips remain visible so the user can remove filters

**Source:** Existing ErrorExplorer.tsx line 258, CONTEXT.md Claude's Discretion (fallback)

## States

| State | Visual Treatment |
|-------|-----------------|
| Card loading | Cards rendered after `useEffect` mount fetch completes. No skeleton -- matches existing pattern (silent load). |
| Card empty (no data for that category) | Card not rendered (`Object.keys(...).length > 0` conditional). Grid adjusts naturally. |

| State | Visual Treatment |
|-------|-----------------|
| Empty (no value) | Browser-native placeholder in dark theme (`color-scheme: dark`) |

| Filled | Value displayed in `text-sf-text` (12px, Inter) |

|-------|-----------------|

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Keyboard navigation | Cards are `<button>` elements (focusable, Enter/Space activates). Datetime inputs are native `<input>` (keyboard-accessible by default). |
| Focus indicator | Global `*:focus-visible` rule (2px solid rgba(99,102,241,0.6), 2px offset) applies to all interactive elements. |
| Screen reader: card items | Button text content includes operation name + count (sufficient for screen reader). |
| Screen reader: period inputs | `<label>` text "Periodo:" associated via wrapping `<div>` structure. Recommend adding `aria-label` to inputs: "Data/hora de inicio" and "Data/hora de fim". |
| Screen reader: filter chips | Button text includes filter label + value. XCircle icon is decorative (`aria-hidden`). |
| Color contrast | sf-text (#e2e8f0) on sf-bg (#0f1117) = 13.5:1 ratio (AAA). sf-textSecondary (#94a3b8) on sf-surface (#1a1d27) = 5.4:1 ratio (AA). sf-primary (#6366f1) on sf-bg (#0f1117) = 4.6:1 ratio (AA). |
| Reduced motion | Global `prefers-reduced-motion` media query in index.css disables animations. |
